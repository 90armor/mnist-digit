# Agent Rules

## Before Coding

1. Read `docs/PROJECT_OVERVIEW.md`
2. Read `docs/ARCHITECTURE.md`
3. Read `docs/COMPONENT_GUIDE.md`

## Rules

- Never create duplicate APIs.
- Follow the existing `train → artifact → infer` architecture.
- Source code goes in `src/`. Documentation goes in `docs/`. Agent config goes in `.ai/`.
- Update `docs/` when changing public interfaces or adding components.
- Ask before any data format or schema changes.
